Mental Health Compact Responsibility App

Admission responsibility checker

Enter the facts as they are known. The app separates provider admission responsibility from ICB/commissioner payment responsibility, because disputes often arise when those two are treated as the same issue.

Core Compact rules for your app

Use these as the rule engine. They are intentionally phrased as decision rules, not as long policy paragraphs.

RuleApp logicPractical finance/dispute point
Provider responsibilityFor adult admissions, start with the Mental Health Trust responsible for the area where the person is usually resident.Do not use GP area alone to decide provider admission responsibility when the Compact points to usual residence.
Residence evidenceIf the person says they live at an address, hostel or temporary residence, accept that as usual residence. Do not require tenancy or utility proof.This prevents delay from unnecessary proof-of-address disputes.
No address / NFAIf no residence is known or given, default to the Trust closest to where the person was assessed. For s136, use where they were first detained.The app should force an answer instead of leaving the case unresolved.
Ongoing care exceptionIf within 6 months of inpatient care, receiving s117 aftercare, or open to a community team for treatment, and no transfer of care has been initiated, the ongoing-care Trust may be responsible if the patient prefers that.This is the key exception to simple residence-based allocation.
Long distance from homeIf transfer to the residence/ongoing-care Trust is not in the person’s best interests at that time, the nearest Trust to assessment should admit temporarily.Temporary admission should not become a finance blocker; later transfer/recharge can be arranged.
Funding separate from admissionCommissioner responsibility is governed by Who Pays principles, usually GP first, then usual residence if no GP.Admission should not be refused or delayed because ICB funding is unclear. Recharging arrangements should follow.
Capacity disputeA Trust should not refuse admission until formal escalation has been enacted and alternatives explored.Capacity pressure is an escalation issue, not an automatic refusal basis.
Final defaultIf a suitable bed cannot be secured even after escalation, the Mental Health Trust closest to where the person was assessed should admit.Use this as the end-point rule to avoid prolonged dispute.

Operational pathway guidance

Use this when building more advanced versions of the app with pathway-specific modules.

15mPolice/LAS should not wait over 15 minutes once HBPoS capacity is confirmed.
1hMental health service response to referral; HBPoS internal escalation begins early.
3hAMHP and s12 doctors should attend within 3 hours unless clinical grounds delay.
4hPhysical and mental health assessment and care plan target.
6hEscalate inpatient admission delay to Surge Services.
12hAdmissions beyond 12 hours require formal review/reporting.
Emergency Department
ED completes preliminary physical assessment, risk assessment and mental state observations. Liaison Psychiatry should be alerted quickly and should respond within one hour of referral. Where MHA assessment is needed, AMHP coordinates; assessment should not wait for bed availability.
Community / section 135
Crisis/community teams assess and involve AMHP where formal assessment is required. If s135 assessment cannot safely occur at home, AMHP arranges conveyance to the closest HBPoS with capacity.
Police custody
Police contact Liaison and Diversion. FMEs/L&D attend, assess health and mental health need, and request AMHP if formal assessment is required. AMHP arranges conveyance if formal admission follows.
Section 136 / HBPoS
Police should contact MHCAL before using s136. The closest HBPoS with capacity should be secured. If closest HBPoS has no capacity, that facility coordinates alternatives and escalates rather than simply refusing.

Deploy this one-file app to Vercel

Option A: Vercel drag-and-drop

  1. Save this file as index.html.
  2. Create a folder named compact-responsibility-app.
  3. Put only this index.html file inside it.
  4. Go to Vercel and import/drag the folder as a static project.
  5. No build command is required. Output directory can be left blank.

Option B: GitHub + Vercel

  1. Create a GitHub repository.
  2. Upload this file as index.html at the repository root.
  3. Import the repository in Vercel.
  4. Framework preset: Other.
  5. Build command: leave empty. Deploy.
